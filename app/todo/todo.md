Contexto del Proyecto:
Actúa como un Arquitecto de Software Senior. Debes diseñar la aplicación "CESIZen", un proyecto simulado para el Ministerio de Salud francés enfocado en la salud mental y gestión del estrés. El objetivo es crear un MVP (Actividad 2 del título CDA) que sea funcional, responsive y "Mobile First".Stack Tecnológico Requerido:Framework: Next.js (App Router) para manejar la Vista (React) y el Controlador (API Routes).Base de Datos: PostgreSQL con Drizzle ORM (para definir el Modelo y el MLD).UI: Tailwind CSS + Shadcn UI (enfocado en accesibilidad y diseño móvil).Autenticación: NextAuth.js o similar para la gestión de cuentas.Testing: Jest para tests unitarios y Playwright para funcionales/regresión.Módulos a Desarrollar (Obligatorios según pliego):Gestión de Cuentas: Registro, login, perfiles (Usuario y Administrador) y recuperación de contraseña.Información: CMS simple para páginas de salud mental y menús dinámicos.Módulo Elegido - Tracker de Emociones: Diario de a bordo con el referente de 2 niveles (ej: Joie -> Fierté) y visualización de reportes semanales/mensuales.Tareas Específicas del Spec:Modelado de Datos (MLD): Genera el esquema de base de datos en TypeScript usando Drizzle. Debe incluir tablas para users, roles, info_pages, emotions_ref (niveles 1 y 2), y user_logs.Arquitectura MVC: Explica cómo la estructura de carpetas de Next.js cumple con el diseño solicitado (Models en Drizzle, Views en componentes, Controllers en API routes).Lógica del Tracker: Implementa la lógica para filtrar y visualizar emociones por períodos (semana, mes, trimestre).Seguridad y RGPD: Implementa hashing de contraseñas (bcrypt), protección CSRF y asegura que no haya transferencia de datos fuera de la UE.Plan de Testing: Crea los archivos de configuración para tests automatizados. Define un escenario de prueba funcional para el login y uno para añadir una emoción al tracker.Entregables esperados en el código:Código fuente estructurado y comentado en francés/inglés.Archivo README.md que sirva como Guía de Instalación.Scripts de automatización de tests.
